指令詳解: `ip addr`

ip addr (或 `ip a`) 是 Linux 系統中一個功能強大的指令，用於顯示和管理網路介面。它是舊版 `ifconfig` 指令的現代替代品，提供更詳細和彈性的網路資訊。

基本用法

要查看所有網路介面的詳細資訊，只需執行：

ip addr

或簡寫為：

ip a

輸出範例解析

執行 ip addr 後，您會看到類似以下結構的輸出（具體內容會因您的系統配置而異）：

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
       valid_lft forever preferred_lft forever
    inet6 ::1/128 scope host 
       valid_lft forever preferred_lft forever
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP group default qlen 1000
    link/ether 00:0c:29:ab:cd:ef brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic eth0
       valid_lft 86242sec preferred_lft 86242sec
    inet6 fe80::a00:27ff:fe8e:e1e1/64 scope link 
       valid_lft forever preferred_lft forever

主要資訊解釋：

  • **`lo` (Loopback 介面)**:

    一個虛擬的環回介面，用於本機內部通訊（例如，程式之間透過 `127.0.0.1` 互相通訊）。它沒有實體網路連接。

    • `inet 127.0.0.1/8`: IPv4 本機位址。
    • `inet6 ::1/128`: IPv6 本機位址。
  • **`eth0` (乙太網路介面)**:

    這是一個典型的實體乙太網路卡介面名稱（在較新的系統中可能是 `enp0s3` 或其他名稱），它連接到您的局域網 (LAN) 或網際網路。

    • `link/ether 00:0c:29:ab:cd:ef`: 顯示該網路卡的 MAC 位址 (Media Access Control Address)，是硬體的唯一識別碼。
    • `inet 192.168.1.100/24`: 這是您的 IPv4 位址。
      • `192.168.1.100`: IP 位址。
      • `/24`: 子網路遮罩 (subnet mask)，表示網路位址佔 24 位元。
      • `brd 192.168.1.255`: 廣播位址。
      • `scope global dynamic`: 表示這是一個可以從外部網路存取、透過 DHCP 動態分配的位址。
    • `inet6 fe80::.../64`: IPv6 位址。
  • **狀態標誌 (如 ``)**:

    表示介面的狀態。`UP` 表示介面已啟用，`LOWER_UP` 表示實體層連接已建立。

  • **`mtu` (Maximum Transmission Unit)**:

    最大傳輸單元，表示單個封包最大可以傳輸的位元組數。

常用選項

  • `ip a show eth0`: 只顯示 `eth0` 介面的資訊。
  • `ip a show up`: 只顯示啟用的 (UP) 介面。
  • `ip -s a`: 顯示更詳細的統計資訊 (statistics)。

通過理解 ip addr 的輸出，您可以快速掌握本機的網路配置情況，這對於網路故障排除和安全檢查至關重要。
