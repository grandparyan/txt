指令詳解: `cat` (查看/串接檔案)

【使用注意】

cat 是為「文字檔案」設計的。如果您不小心 cat 了一個「二進位檔」（例如一張圖片、一個程式或一個 .zip 壓縮檔），它不會損壞檔案，但會在您的終端機上印出大量無法閱讀的亂碼，甚至可能卡住您的終端機視窗。

在安全情境下，請務必遵守權限：**讀取您沒有權限的檔案（例如他人的私鑰或系統密碼檔）是未經授權的存取行為。**

cat (來自 "concatenate"，意為「串接」) 是 Linux 中最常用的指令之一。它最廣為人知的用途是**將一個或多個檔案的內容印到螢幕上**。

1. 基本用法：查看單一檔案

這是 cat 最常見的用法。在您用 ls 看到一個 `README.md` 檔案後，您可以使用 cat 來閱讀它。

cat README.md

輸出：

# 歡迎使用我的專案

這是一個範例 README 檔案。
`cat` 指令會將這些文字完整地顯示在您的終端機上。

2. cat 的原意：串接檔案

cat 的真正強大之處在於它可以將多個檔案「串接」在一起。搭配 > (重導向) 符號，您可以將多個檔案合併成一個新檔案。

# 將 part1.txt 和 part2.txt 的內容合併，並存成 new_file.txt
cat part1.txt part2.txt > new_file.txt
  • part1.txt: "這是第一行。"
  • part2.txt: "這是第二行。"
  • 執行後，new_file.txt 的內容將會是 "這是第一行。這是第二行。"
  • >: 這個符號會**覆蓋** (Overwrite) new_file.txt 的內容。如果您想「附加」(Append) 到檔案末尾，請使用 >>。

3. 快速建立檔案

您可以使用 cat 搭配 > 來快速建立一個小檔案，而不需要打開完整的文字編輯器（如 `nano` 或 `vim`）。

cat > my_note.txt

執行此指令後，終端機看起來會像在「等待」。您現在可以開始打字：

這是我要記下的第一行筆記。
這是第二行。
(接著按下 Ctrl+D 組合鍵)
  • 當您輸入完畢後，按下 Ctrl+D (代表 End-of-File)，cat 就會停止接收輸入，並將您剛才輸入的內容儲存到 `my_note.txt` 中。

4. 搭配管線 (Pipe) 使用

cat 經常作為「管線」(|) 的開頭，將檔案內容傳送給其他指令進行處理。

# 在 access.log 檔案中，搜尋 (grep) 所有包含 "ERROR" 的行
cat access.log | grep "ERROR"
  • cat access.log: 讀取整個日誌檔案。
  • |: 將左邊指令的「輸出」變成右邊指令的「輸入」。
  • grep "ERROR": 在收到的輸入中，只篩選出包含 "ERROR" 的行。
