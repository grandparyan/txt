指令詳解: `mail` (系統通知)

【極度危險警告】嚴禁資料外洩

mail 指令或 mailto: 協定本身是中性的，但它們可能被用於惡意目的。

在未經授權的情況下，將您透過 `cat` 或其他方式**竊取**來的資料（例如設定檔、密碼檔、私鑰）傳送到外部電子郵件地址，被稱為「資料外洩」(Data Exfiltration)。

這是一種**嚴重的犯罪行為**，會導致極其嚴重的法律後果。本頁面僅介紹 `mail` 指令的合法管理用途。

mail 是 Linux 系統上的一個經典工具程式（通常由 `mailutils` 套件提供），它允許使用者從指令列收發電子郵件。在現代系統管理中，它最常用於**自動化腳本**，以便在任務完成或發生錯誤時向管理員傳送通知。

（注意：要讓 mail 指令能真正寄信到外部（如 Gmail），伺服器本身必須經過繁瑣的設定（例如 Postfix 或 Sendmail），並處理 SPF/DKIM 等反垃圾郵件機制。以下範例假設伺服器已設定好本地郵件傳遞。）

合法用途 1：傳送簡單通知

這是最常見的用法。您寫了一個腳本，並希望在腳本執行完畢時收到通知。

# 語法: echo "[郵件內容]" | mail -s "[郵件主旨]" [收件人]
echo "備份腳本已於 $(date) 成功完成。" | mail -s "伺服器 A 備份完成" admin@my-company.com
  • echo "...": 產生您要傳送的郵件內容。
  • |: 將內容「傳送」給 mail 指令。
  • -s "...": 指定郵件的**主旨 (Subject)**。
  • admin@my-company.com: 您（系統管理員）的**合法**電子郵件地址。

合法用途 2：將日誌檔案作為附件傳送

假設您想每天檢查一次系統日誌。您可以使用 mail 將**您有權存取**的日誌檔案作為通知寄給自己。

# 語法: mail -s "[主旨]" -A [要附加的檔案] [收件人] < [包含郵件內文的檔案]
mail -s "每日系統日誌" -A /var/log/syslog admin@my-company.com < /dev/null
  • -A /var/log/syslog: **附加 (Attach)** syslog 檔案。這是您作為管理員有權查看的檔案。
  • < /dev/null: 由於我們只想傳送附件，不需要內文，因此我們從 /dev/null（一個「空」設備）讀取內文，表示郵件內文為空。

關於 `mailto:`

您在問題中提到的 `mailto:` 是一個「URI 協定」，而不是一個 Linux 指令。它主要用於 **HTML** 和網頁瀏覽器。

例如，在網頁上：

<a href="mailto:support@example.com">聯絡我們</a>

當使用者點擊這個連結時，瀏覽器會嘗試**打開使用者本機的預設郵件客戶端**（例如 Outlook, Apple Mail 或 Thunderbird），並自動填入收件人地址。
