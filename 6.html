指令詳解: `ssh` (安全遠端登入)

【核心安全工具】

ssh (Secure Shell) 是用於**防禦**網路竊聽的標準工具。它取代了古老且不安全的 telnet 和 rlogin。

使用 ssh 可以確保您的登入憑證和所有操作都在加密通道中傳輸，使 `tcpdump` 等監聽工具**無法**竊取您的明文密碼。

ssh 是您作為系統管理員、開發人員或安全專業人員日常工具箱中最重要的工具之一。它允許您從您的本機終端機，安全地存取並操作一台遠端伺服器。

基本用法：密碼登入

這是最基礎的登入方式。您需要知道三件事：

  • 遠端主機的 IP 位址或網域名稱 (例如 `192.168.1.100` 或 `myserver.com`)
  • 您在該主機上的**合法使用者名稱** (例如 `admin` 或 `devuser`)
  • 該使用者的密碼
ssh devuser@192.168.1.100
  • devuser: 您的使用者名稱。
  • 192.168.1.100: 伺服器的 IP 位址。

【重要】首次連線指紋驗證

當您第一次連線到某台主機時，您會看到這樣的提示：

The authenticity of host '192.168.1.100 (192.168.1.100)' can't be established.
ECDSA key fingerprint is SHA256:l2S...[一長串指紋]...E.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
  • **這是什麼？** 這是伺服器的「公鑰指紋」，就像是伺服器的身分證。
  • **您該怎麼做？** 在一個安全的網路環境中（例如您確定您連的是對的伺服器），輸入 `yes`。這會將伺服器的公鑰儲存到您本機的 `~/.ssh/known_hosts` 檔案中。
  • **安全機制**：如果下次您再連線同一台伺服器，但指紋**變了**，`ssh` 會**立即警告您並拒絕連線**。這可能表示伺服器重裝了，但也可能是您正在遭受「中間人攻擊」！

輸入 `yes` 後，系統會提示您輸入 `devuser` 的密碼。輸入正確後，您就成功登入了。

推薦用法：公鑰認證 (免密碼)

依賴密碼並不安全（密碼可能被猜到、或在您使用不安全的 `http` 服務時被竊取）。最強大、最推薦的 `ssh` 登入方式是使用「金鑰對」。

這就像是為您的電腦製作一把專屬的「數位鑰匙」，並將「鎖」
