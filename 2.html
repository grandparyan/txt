指令詳解: `nmap`

【極度重要】法律與道德警告

Nmap 是一個功能強大的網路掃描工具。它既是網路管理員的得力助手，也是攻擊者常用的偵察工具。

您【絕對禁止】在未經明確授權的情況下對任何不屬於您的網路或主機使用 Nmap。

未經授權的掃描在多數國家和地區均屬**違法行為**，可能導致嚴重的法律後果。本教學僅限於**教育目的**和在**您自己的網路或授權的測試環境**中使用。

Nmap (Network Mapper) 是一個開源的網路探測和安全稽核工具。系統管理員使用它來執行網路清點、管理服務升級排程，以及監控主機或服務的運作狀態。

合法與防禦性用途

作為一個負責任的系統管理員或安全學習者，nmap 主要用於「防禦」目的：

  • 網路資產盤點: 快速找出在您**自己**的網路上有哪些設備在線。
  • 防火牆規則驗證: 檢查您**自己**的伺服器或防火牆是否正確關閉了不應開放的埠。
  • 服務檢查: 確認您**自己**的伺服器上，預期要運行的服務 (如網頁伺服器、SSH) 是否正常啟動並在監聽。

常用防禦性指令

1. 主機發現 (Ping Scan)

這是最安全、最不具侵擾性的掃描。它只會檢查在您指定的網路範圍內，有哪些主機是在線的 (回應 Ping)，而不會嘗試掃描它們的埠。

這對於清點您家中或公司內部網路 (您有權管理) 有多少設備非常有用。

nmap -sn 192.168.1.0/24
  • -sn: (Skip Port Scan) 告訴 Nmap 不要進行埠掃描，只做主機發現。
  • 192.168.1.0/24: 您要掃描的子網路範圍 (請替換為您**自己**的網路範圍，例如 `192.168.0.0/24`)。

2. 掃描單一主機的常用埠

這用於檢查**您自己**的特定主機（例如您架設的網站伺服器或 `localhost`）開放了哪些常見的服務埠。

nmap 192.168.1.100

或者，掃描您自己的本機：

nmap localhost
  • nmap 預設會掃描目標主機上 1000 個最常見的 TCP 埠。

如何解讀輸出

Starting Nmap 7.91 ( https://nmap.org ) at 2025-11-15 02:30 CST
Nmap scan report for myserver.local (192.168.1.100)
Host is up (0.0010s latency).
Not shown: 997 closed ports
PORT    STATE SERVICE
22/tcp  open  ssh
80/tcp  open  http
443/tcp open  https

Nmap done: 1 IP address (1 host up) scanned in 0.45 seconds
  • `PORT 22/tcp open ssh`:

    表示 SSH 服務 (安全遠端登入) 正在 22 埠運行。如果您不打算開放遠端登入，這就是一個**安全警示**，您應該去檢查防火牆規則。

  • `PORT 80/tcp open http`:

    表示標準的網頁伺服器 (HTTP) 正在 80 埠運行。如果您正在架設網站，這是正常的。

  • `PORT 443/tcp open https`:

    表示安全的網頁伺服器 (HTTPS) 正在 443 埠運行。

  • `Not shown: 997 closed ports`:

    非常好！這表示其他 997 個常見埠都是關閉的，有效降低了受攻擊面。

總結來說，nmap 是一個強大的稽核工具。請務必在**授權範圍內**負責任地使用它，以強化您自己的網路安全。
